# 운영 대시보드 메뉴얼

관리자 **운영 대시보드**(`/admin` → Dashboard)에 표시되는 각 요소의 의미·집계 기준·주의사항입니다.

데이터는 서버에서 Supabase를 조회해 **페이지 로드 시점**에 집계합니다. (실시간 스트리밍·자동 갱신 아님)

**관련 코드**

| 역할 | 파일 |
|------|------|
| 집계 로직 | `src/lib/admin-dashboard-data.ts` |
| UI·라벨 | `src/components/admin/AdminDashboardPanel.tsx` |
| 채팅 LLM 오류 로그 | `src/lib/llm-error-log.ts`, `supabase/migrations/20260517180000_llm_error_events.sql` |

## 1. 공통 규칙

| 항목 | 내용 |
|------|------|
| **기준 시각** | 서버 `Date` 기준 **오늘 00:00**을 “오늘” 시작으로 사용 |
| **어제** | 어제 00:00 ~ 오늘 00:00 (전일) |
| **최근 7일 / 30일** | 오늘 00:00을 끝으로 **역산** (달력 주·월 아님) |
| **조회 범위** | 대부분 **최근 30일** 데이터만 로드 (`orders` 2000건, `voice_sessions` 2000건 등 상한 있음) |
| **더미 데이터** | 없음. DB가 비어 있으면 **0** |
| **기간 토글 (어제 / 7일 / 30일)** | **차트·미니 KPI·캐릭터·순위·퍼널**만 변경 |
| **상단 6+6 KPI** | 상태 6개는 고정. 운영 6개(매출·활성·가입·충전·오류)는 **어제/7일 고정** — 토글과 무관 |

## 3. 첫 번째 줄 KPI (상태 6개)

DB·스키마 **연결·건수 스냅샷**입니다. PG·LLM 실시간 헬스체크가 **아닙니다**.

| 라벨 | 숫자 의미 | DB / 로직 |
|------|-----------|-----------|
| **점사 메뉴 상품** | 점사 메뉴로 분류되는 상품 slug 개수 | `products` 중 `isFortuneMenuCatalogProductSlug(slug)` 인 행 |
| **캐릭터** | 등록 캐릭터 수 | `characters` `count` |
| **리뷰 행** | reviews 테이블 **전체 행 수** | `reviews` (공개/쇼케이스 여부 무관) |
| **결제 테이블** | `조회 OK` / `오류` | `payments` 1건 조회 성공 여부 |
| **음성 테이블** | `조회 OK` / `오류` | `voice_sessions` 조회 성공 여부 |
| **점사 테이블** | `조회 OK` / `오류` | `fortune_requests` 조회 성공 여부 |

## 4. 두 번째 줄 KPI (운영 6개) — 기간 토글 **미적용**

### 4-1. 어제 결제 매출

- **값**: 어제 `orders` 중 `status = 'paid'` 인 주문의 `amount_krw` **합계**
- **하단 %**: **그저께** 동일 조건 합계 대비 증감 (“전일 대비”)
- **포함**: 크레딧 패키지·점사 상품 등 **모든 paid 주문**
- **미포함**: `pending`, `cancelled` 등

### 4-2. 최근 7일 결제 매출

- **값**: 오늘 00:00 기준 **최근 7일** paid 주문 `amount_krw` 합계
- **하단 %**: **그 직전 7일** (8~14일 전 구간) 대비 (“직전 7일 대비”)
- **주의**: “이번 주 월~일” 같은 **달력 주간 매출이 아님**

### 4-3. 어제 활성 user_ref

- **값**: 어제 아래 3종 테이블에 `user_ref`가 **비어 있지 않은** 행을 모아 **중복 제거한 개수**
  - `voice_sessions` (`started_at`)
  - `fortune_requests` (`created_at`)
  - `text_chat_sessions` (`started_at`)
- **같은 사람이 음성+점사+채팅을 모두 하면**: 서비스별 prefix(`v:`, `f:`, `c:`) 때문에 **최대 3까지** 잡힐 수 있음
- **일반 DAU/MAU 아님**: 로그인 UUID만 세지 않음. `guest`, `visitor_xxx`도 각각 1명으로 셈
- **하단 %**: 그저께 동일 로직 대비

#### user_ref란?

주문·세션에 붙는 **문자열 이용자 식별자**입니다.

| 경우 | 예시 |
|------|------|
| 로그인 회원 | Supabase Auth `user.id` (UUID) |
| 비로그인 | `guest`, `visitor_<uuid>` (브라우저 localStorage) |

회원 마스터(`yeonun_social_users`)와 **별개** 필드입니다.

### 4-4. 어제 소셜 가입

- **값**: 어제 생성된 **소셜 계정** 수
- **DB**: `yeonun_social_users` (`deleted_at IS NULL`, `created_at` 어제 구간)
- **하단 %**: 그저께 가입 수 대비

### 4-5. 어제 크레딧 충전

- **값**: 어제 paid 주문 중 `product_slug`가 `credit-package-`로 시작하는 **주문 건수**
- **보조 문구**: 해당 주문들의 `amount_krw` 합 (원화 충전액)
- **의미**: 크레딧 패키지 **결제 완료** 건수·금액

### 4-6. 어제 처리 오류

| 구분 | 집계 조건 | 테이블 |
|------|-----------|--------|
| **음성** | 어제 시작 + `status = 'error'` | `voice_sessions` |
| **점사** | 어제 생성 + `status = 'failed'` | `fortune_requests` |
| **채팅** | 어제 `llm_error_events` where `service = 'chat'` | `llm_error_events` |
| **합계** | 위 세 가지 합 | |

**채팅 오류**는 다음 때 `llm_error_events`에 기록됩니다.

- `/api/chat/consult-stream` upstream 실패
- 채팅 UI 스트림 실패 시 `/api/chat/llm-error` 호출

**미집계**: 앱 채팅 상담 중 **localStorage만** 쓰는 세션(DB `text_chat_*` 미저장)의 오류는 여기에 안 잡힐 수 있음.

> `llm_error_events` 마이그레이션(`20260517180000_llm_error_events.sql`)이 DB에 없으면 채팅 오류는 **0**으로 표시됩니다.

## 5. 매출 · 이용 현황 (기간 토글 **적용**)

### 5-1. 결제 매출 차트

| 토글 | 차트 제목 | 일수 |
|------|-----------|------|
| 어제 | 7일 결제 매출 (어제 포함) | 최근 7일 일별 |
| 7일 | 7일 결제 매출 | 7일 |
| 30일 | 30일 결제 매출 | 30일 |

- **Y축**: 해당 일 `paid` 주문 `amount_krw` 합
- **X축**: 날짜 라벨 (마지막 날 “오늘”)
- 마지막 점 툴팁: 해당 일 매출 원화

### 5-2. 미니 4칸 (토글 기간 기준)

| 라벨 | 의미 | DB |
|------|------|-----|
| **음성 상담 시간** | `voice_sessions.duration_sec` 합 ÷ 3600 (시간, 소수 1자리) | 기간 내 세션 |
| **채팅 메시지 수** | `text_chat_messages` **행 수** | 기간 내 `created_at` |
| **점사 요청 수** | `fortune_requests` **행 수** | 기간 내 (status 무관) |
| **소셜 가입** | `yeonun_social_users` 신규 가입 수 | 기간 내 |

**주의**

- “채팅 메시지 수”는 DB `text_chat_messages` 기준.
- 모달 채팅 상담이 **로컬만** 저장되면 이 숫자에 **포함되지 않을 수 있음**.
- 아래 **캐릭터별 이용 비율**의 “채팅”은 **메시지 수가 아니라 세션 건수**입니다.

### 5-3. 캐릭터별 이용 비율

부제: **음성 세션·점사 요청·채팅 세션 건수 합산**

| 소스 | 테이블 | 캐릭터 키 |
|------|--------|-----------|
| 음성 | `voice_sessions` | `character_key` |
| 점사 | `fortune_requests` | `product_slug` → `charKeyFromSlug` 추정 |
| 채팅 | `text_chat_sessions` | `character_key` |

- **건수**: 기간 내 위 3종 각 1건씩 합산 (캐릭터별)
- **%**: 캐릭터별 건수 비율 (반올림)
- **미포함**: localStorage 전용 채팅 상담 세션

### 5-4. 소셜 계정 누적 (provider)

- **도넛 중앙**: `yeonun_social_users` 전체 수 (삭제 제외)
- **비율**: Google / Kakao / Naver `provider` 비율
- **기간 토글 무관** — **전체 누적**

## 6. 콘텐츠 · 전환 (기간 토글 **적용**)

### 6-1. 결제 상품 순위 Top 5

- **건수**: 기간 내 `paid` 주문, `product_slug`별 **주문 건수**
- **제외**: `credit-package-*` (크레딧 충전)
- **표시**: 상품명, 캐릭터·가격 메타, 건수

### 6-2. 리뷰 별점 분포

- **대상**: `reviews` 전체 (기간 필터 **없음**)
- **평균**: 별점 1~5만 반영한 산술 평균
- **막대**: ★5 ~ ★1 각 건수 (막대 길이는 최대 구간 대비 상대값)
- **상단 “리뷰 행”**과 달리, 별점 분포는 **유효 별점(1~5) 행**만 평균·분포에 사용

### 6-3. 기간별 전환 (참고)

마케팅 퍼널이 아니라 **서로 다른 단위를 나란히 둔 참고 차트**입니다.

| 단계 | 값의 의미 | 단위 |
|------|-----------|------|
| **소셜 계정 누적** | `yeonun_social_users` 전체 | 계정 수 |
| **기간 소셜 가입** | 토글 기간 신규 가입 | 명 |
| **음성 세션 + 점사 요청** | 두 테이블 **이벤트 건수 합** | **건** (사용자 수·채팅 미포함) |
| **결제 완료 주문** | 기간 내 `paid` **주문 건수** | 건 |

막대 너비·% 문구는 **누적 계정 수 대비** 등 참고용이며, 엄밀한 전환율 분석용이 아닙니다.

## 7. 운영 알림

자동 생성 **요약 카드**입니다.

| 조건 | 톤 | 예시 |
|------|-----|------|
| 어제 `fortune_requests` failed ≥ 1 | warn | 점사 요청 failed N건 |
| reviews 중 ★1~2 **누적** ≥ 1 | warn | 낮은 별점 후기 |
| 어제 매출 > 그저께 | ok | 어제 결제 매출 전일 대비 +N% |
| 위 없음 | ok | 특이 알림 없음 |

## 8. 자주 헷갈리는 점 (FAQ)

1. **상단 매출 KPI vs 차트**  
   - 상단: 어제 / 최근 7일 **고정**  
   - 차트: 토글에 따라 7일 또는 30일

2. **“명” vs “건”**  
   - 소셜 가입·user_ref 활성: **명(식별자 개수)**  
   - 음성·점사·주문·메시지·세션: 대부분 **건(이벤트 수)**

3. **채팅 관련 지표 3종**

   | UI | 집계 |
   |----|------|
   | 채팅 메시지 수 (미니 KPI) | `text_chat_messages` 행 수 |
   | 캐릭터별 이용 (채팅) | `text_chat_sessions` 세션 건수 |
   | 어제 활성 user_ref (채팅) | `text_chat_sessions.user_ref` |

4. **리뷰 행(상단) vs 리뷰 별점(하단)**  
   - 상단: 테이블 **행 전체**  
   - 하단: **별점 1~5인 행**만 평균·분포

5. **처리 오류 ≠ CS 티켓 수**  
   - DB 상태·오류 로그 기준이며, 사용자 문의 건수와 1:1이 아님

6. **캐릭터별 이용 vs 퍼널 “음성+점사”**  
   - 캐릭터 차트: **채팅 세션 포함**  
   - 퍼널 3단계: **채팅 미포함** (음성 세션 + 점사 요청만)
